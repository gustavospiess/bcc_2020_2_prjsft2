##2020-10-09 - aula08 desenvolvimento do projeto
###Relatar O QUE FOI FEITO:

- Identificada a inviabilidade do `DjangoBuilder`

Utilizando o `DjangoBuilder`, não foi possível fazer a execução do mesmo
localmente de forma estável. Por conta de dependências, não estava sendo uma
solução aceitável para o desenvolvimento dos formulários.

- Identificada como tecnologia para o desenvolvimento dos formulários o
    `DjangoCrispyForms`.

Foi identificada essa alternativa para o desenvolvimento dos formulário,
utilizado um conjunto de formulários de exemplo para desenvolver os formulários
seguintes.

- Desenvolvido o formulário de entrada dos parâmetros gerais do algoritmo.

Utilizando a ferramenta identificada, foi desenvolvido o formulário de entrada
de dados, algumas validações que seria interessantes não foram possíveis de
serem desenvolvidas, mas não são critérios obrigatórios, os filtros de mínimo e
máximo foram inseridos de forma adequada.

- Pesquisado para o desenvolvimento do formulário dinâmico de cadastro de
    características.

Realizada pesquisa de como utilizar sub formulários no Django para realizar o
desenvolvimento de um formulário dinâmico. Não foi identificada uma solução
viável e portanto mais pesquisa será necessária.
